## Components

### `Navbar`

- Sticky, semi-transparent header with backdrop blur.
- Contains brand logo link (left), nav links (centre/hidden on mobile), and a CTA button (right).
- **Customise:** Replace `YourBrand` with your brand name; add/remove nav links.

### `Hero`

- Full-width centred section with a badge, headline, sub-headline, and two CTA buttons.
- **Customise:** Edit the headline copy and button `href` values.

### `Features`

- 3-column responsive grid of feature cards (icon + title + description).
- Data is stored in a `features` array inside the component вЂ” edit that array to change cards.
- **Customise:** Add or remove items from the `features` array.

### `CTA`

- Indigo banner with a headline, supporting copy, and two action buttons (GitHub + Vercel deploy).
- **Customise:** Update `href` values to point to your repository and deployment URL.

### `Footer`

- Single-row footer with copyright text (auto-updated year) and external links.
- **Customise:** Replace links with relevant ones for your product.

---

## Conventions

- **Components** are PascalCase, one per file, default-exported.
- **Import alias** `@/*` resolves to `src/*` (configured in `tsconfig.json`).
- **Colours** use Tailwind CSS semantic palette вЂ” swap `indigo` for any other Tailwind colour to re-theme.
- **No global state** вЂ” all sections are stateless server components.

---

## Adding a new section

1. Create `src/components/NewSection.tsx`.
2. Export a default React function component.
3. Import and render it inside `src/app/page.tsx` between the desired sections.

## Deployment

The fastest way to deploy is **Vercel**:

```
https://vercel.com/new/clone?repository-url=https://github.com/mrpavlitzky/landing-next-template
```

Alternatively, run `npm run build && npm run start` on any Node.js 18+ host.
